# Diagrama de Clases

Este diagrama muestra cómo está organizado el sistema por dentro: qué información maneja cada parte, cómo se conectan entre sí y qué reglas siguen. Está dividido en tres partes para que sea más fácil de leer.

La primera parte muestra a los actores del sistema: los usuarios, jugadores, capitanes, árbitros, organizadores y administradores, y cómo se agrupan en equipos. La segunda parte muestra el corazón del torneo: cómo funcionan los torneos, partidos, pagos y alineaciones. La tercera parte muestra los patrones de diseño que se usaron para manejar los cambios de estado y las notificaciones, y cómo funciona la seguridad con JWT y OAuth2.
